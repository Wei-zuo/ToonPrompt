# 剧本创作

当用户已经确认了一句话故事，或明确要求“继续写成剧本”“扩成完整剧本”“给我人物和场景”时，读这个文件。

## 1. 剧本轮必须完成什么

默认至少完成五件事：

1. 理解输入中的核心冲突、情绪内核和人物关系
2. 扩写为一个完整剧本
3. 生成完整剧情
4. 生成人物设定
5. 生成场景列表

不要只给空洞提纲，也不要只给设定页。

## 2. 默认交付结构

当用户没有指定格式时，剧本轮默认包含：

- `opening`：故事开端
- `development`：故事发展
- `climax`：故事高潮
- `ending`：故事结尾
- `full story / full script`：完整剧情或剧本文本
- `characters`：人物设定
- `scenes`：场景列表

如果题材更实验、反传统或故意碎片化，可以变动这个骨架，但必须让读者仍然能看出：

- 故事怎么开始
- 关系怎么升级
- 冲突怎么翻面
- 最后留下什么结果或代价

## 3. 剧本创作的默认顺序

### A. 先读懂原始故事核心

先明确：

- 核心冲突是什么
- 情绪内核是什么
- 谁和谁之间最重要
- 这个故事真正卖的不是设定，而是什么关系 / 欲望 / 代价

如果这些都没抓住，就不要急着扩写。

### B. 先搭完整剧情

完整剧情至少要包含：

- 起点状态
- 触发事件
- 关系推进
- 冲突升级
- 中段翻面
- 高潮选择或反噬
- 结尾结果

不要把完整剧情写成几句概括，要让读者能真正“看见它怎么演”。

### C. 再补人物设定

每个主要角色，至少要交代：

- 身份
- 目标
- 内在匮乏
- 核心矛盾
- 和其他角色的关系
- 变化弧线

人物设定必须直接服务剧情，不能像人物百科。

### D. 再写场景列表

每场至少明确：

- 场景名称
- 时间 / 地点
- 出场人物
- 本场目标
- 核心事件
- 情绪变化
- 对整体剧情的作用

场景列表要推动故事，不是把已有剧情拆碎重说一遍。

## 4. 写作原则

1. 一切内容服务于原始故事核心，不跑题。
2. 剧本必须完整，不写成空洞提纲。
3. 角色必须有明确目标、矛盾与关系。
4. 场景必须推动剧情或深化人物。
5. 台词和事件要有戏剧性、画面感和完成度。
6. 能用动作表达的，不要全靠解释。
7. 暗线要融入剧情，不要直接说教。
8. 输出内容要便于后续继续拆分为分镜、拍摄脚本或角色设定。

## 5. opening / development / climax / ending 的工作定义

### opening

- 建立人物当前状态
- 建立关系和缺口
- 建立故事的情绪承诺
- 抛出触发事件或异常

### development

- 让人物开始行动
- 让关系发生推进或撕裂
- 让信息、风险、代价不断上升
- 至少出现一次让故事重新理解自己的翻面

### climax

- 逼迫主角做无法回避的选择
- 让主角真正付出代价，或真正完成变化
- 不要只是把设定说出来

### ending

- 收束结果
- 兑现代价
- 留下最终画面、最终关系状态、或最终认知反弹
- 如果题材允许开放式结尾，也必须让开放本身有意义

## 6. 输出时的常见错误

避免：

- 只有 worldbuilding，没有人物推进
- 把剧情写成说明文
- 人物目标模糊，导致冲突飘
- 场景只是信息搬运
- 高潮只是揭示，不是选择
- 结尾只停在概念，没有画面或结果

## 7. 面向后续制作

写完后，默认自检：

- 这份输出能不能继续拆分为分镜
- 能不能交给导演或剪辑继续理解
- 人物有没有明确可演的行为
- 每场戏有没有明确的动作和转折

如果不能继续往下拆，说明剧本轮还不够完成。
